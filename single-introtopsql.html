Introduction to PostgreSQL.

  • By Nishant Ghayal
  • March 29, 2020
  • PostgreSQL

PostgreSQL is a powerful, open-source object-relational database system that uses and extend the SQL language combined with many features that safely store and scale the most complicated data workloads. An object-relational database system means is a database management system similar to a relational database, but with an object-oriented database model: objects, classes, and inheritance are directly supported in database schemas and in the query language.

Why Use PostgreSQL?

PostgreSQL comes with many features when building scaled applications while providing data integrity and fault tolerance. Best part is that it is a widely support open source relational database for small or large datasets. It easily installed, it can provide quick upstart for companies needing a cost saving database platform.

It supports many data types, data integrity, concurrency, disaster recovery, security, and extensibility. For example, you can build a active cluster with terabytes of complex data with replication failover scenerios to keep the database highly available in many Cloud regions. You will then be tolerant to outages in your chosen cloud regions.

History

In 1986 the first implementation was called the "Berkeley Postgres Project", and has gone through many iterations following the paces of IT. In 1994 the technology was renamed to Potgres95 and enhanced with performance and maintainbility improvements. In 1996 it was finally renamed to PostgreSQL as Postgres95 would not progres time. Many in the open source commit changes to improve the source code.
